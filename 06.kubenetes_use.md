|                     | kubectl config view                              | kubeconfig 확인                |
|                     | kubectl config current-context                   | 현재 사용중인 클러스터         |
| 리소스 조회         | kubectl get pods                                 | po                             |
|                     | kubectl get deployment                           | deploy                         |
|                     | kubectl get services                             | svc                            |
|                     | kubectl get ConfigMap                            | cm                             |
|                     | kubectl get secret                               |                                |
|                     | kubectl get cronjob                              |                                |
|                     | kubectl get jobs                                 |                                |
|                     | kubectl get nodes                                |                                |
|                     | kubectl get namespaces                           | ns                             |
|                     | kubectl get events                               |                                |
| 리소스 조회(조건)   | kubectl get pods -n kube-system                  | namespace 지정                 |
|                     | kubectl get pods --all-namespaces                | 모든 namespace 지정            |
|                     | kubectl get pods -l app=nginx                    | 라벨로 조회                    |
|                     | kubectl get pods -w                              | 실시간 조회                    |
|                     | kubectl get pods -o wide                         | 상세 정보                      |
| 리소스 상세 조회    | kubectl describe pod <pod-name>                  |                                |
| 리소스 생성         | kubectl apply -f deployment.yaml                 | yaml로 생성                    |
|                     | kubectl create -f pod.yaml                       | 생성                           |
|                     | kubectl run nginx --image=nginx                  | 간단한 pod 생성                |

|                     | kubectl logs <pod> -c <container>                | 컨테이너 지정                  |
| Pod 접속            | kubectl exec -it <pod-name> -- /bin/bash         | Pod 접속                       |
| 파일 복사           | kubectl cp file.txt pod:/tmp/file.txt            | 파일 복사                      |
| 리소스 상태 확인    | kubectl top nodes                                |                                |
| namespace 관리      | kubectl create ns dev                            | 생성                           |
|                     | kubectl delete ns dev                            | 삭제                           |

|                     | kubectl label pod nginx env-                     | 라벨 삭제                      |

## 로드밸런서

1. L4 로드밸런서: IP 주소나 포트 번호 등을 이용해 트래픽 분산
2. L7 로드밸런서: 애플리케이션 계층을 기반으로 로드를 분산하기 때문에 URL, HTTP 헤더, 쿠키 등과 같은 사용자의 요청을 기준으로 트래픽 분산

### error
```
429 Too Many Requests - Server message: toomanyrequests: You have reached your unauthenticated pull rate limit. https://www.docker.com/increase-rate-limit
```
원인:   
```
메시지에 표시된 것처럼 도커 로그인을 하지 않은 상태(Unauthenticated)에서 너무 많은 컨테이너 이미지를 다운로드(Pull)하여 도커 허브의 일일 다운로드 제한(IP당 6시간 내 100회)을 초과
```
해결책:  
```sh
#도커 허브(Docker Hub) 계정으로 로그인하는 것입니다. 로그인만 해도 제한이 6시간당 200회로 늘어남
docker login
```
