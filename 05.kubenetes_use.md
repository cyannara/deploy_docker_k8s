## 쿠버네티스 명령어

| 카테고리            | 명령                                             | 설명                           |
| :------------------ | :----------------------------------------------- | :----------------------------- |
| 클러스터 정보확인   | kubectl cluster-info                             | 클러스터 API 서버 정보         |
|                     | kubectl version                                  | kubectl / Kubernetes 버전 확인 |
|                     | kubectl config view                              | kubeconfig 확인                |
|                     | kubectl config current-context                   | 현재 사용중인 클러스터         |
| 리소스 조회         | kubectl get pods                                 | po                             |
|                     | kubectl get deployment                           | deploy                         |
|                     | kubectl get services                             | svc                            |
|                     | kubectl get ConfigMap                            | cm                             |
|                     | kubectl get secret                               |                                |
|                     | kubectl get cronjob                              |                                |
|                     | kubectl get jobs                                 |                                |
|                     | kubectl get nodes                                |                                |
|                     | kubectl get namespaces                           | ns                             |
|                     | kubectl get events                               |                                |
| 리소스 조회(조건)   | kubectl get pods -n kube-system                  | namespace 지정                 |
|                     | kubectl get pods --all-namespaces                | 모든 namespace 지정            |
|                     | kubectl get pods -l app=nginx                    | 라벨로 조회                    |
|                     | kubectl get pods -w                              | 실시간 조회                    |
|                     | kubectl get pods -o wide                         | 상세 정보                      |
| 리소스 상세 조회    | kubectl describe pod <pod-name>                  |                                |
| 리소스 생성         | kubectl apply -f deployment.yaml                 | yaml로 생성                    |
|                     | kubectl create -f pod.yaml                       | 생성                           |
|                     | kubectl run nginx --image=nginx                  | 간단한 pod 생성                |
| 리소스 수정         | kubectl edit deployment nginx                    | 설정 수정                      |
|                     | kubectl scale deployment nginx --replicas=3      | 스케일 조정                    |
| 리소스 삭제         | kubectl delete pod nginx                         |                                |
|                     | kubectl delete -f deployment.yaml                | yaml 삭제                      |
| 로그 확인           | kubectl logs <pod-name>                          | 로그                           |
|                     | kubectl logs -f <pod-name>                       | 실시간 로그                    |
|                     | kubectl logs <pod> -c <container>                | 컨테이너 지정                  |
| Pod 접속            | kubectl exec -it <pod-name> -- /bin/bash         | Pod 접속                       |
| 파일 복사           | kubectl cp file.txt pod:/tmp/file.txt            | 파일 복사                      |
| 리소스 상태 확인    | kubectl top nodes                                |                                |
| namespace 관리      | kubectl create ns dev                            | 생성                           |
|                     | kubectl delete ns dev                            | 삭제                           |
| 라벨 관리           | kubectl label pod nginx env=dev                  | 라벨 추가                      |
|                     | kubectl label pod nginx env-                     | 라벨 삭제                      |
| rollout (배포 관리) | kubectl rollout status deployment nginx          | 상태 확인                      |
|                     | kubectl rollout restart deployment nginx         | 재시작                         |
|                     | kubectl rollout undo deployment nginx            | 롤백                           |
|                     | kubectl create job test --from=cronjob/mycronjob | CronJob으로 Job 실행           |
| 디버깅 명령         | kubectl get events                               | 이벤트 확인                    |
|                     | kubectl describe pod <pod>                       | pod 문제 확인                  |
|                     |                                                  |                                |
|                     |                                                  |                                |

가장많이 쓰는 명령

```sh
kubectl get pods
kubectl get pods -A
kubectl get pods -o wide
kubectl describe pod <pod>
kubectl logs -f <pod>
kubectl exec -it <pod> -- bash
kubectl apply -f file.yaml
kubectl delete -f file.yaml
kubectl get events
kubectl rollout restart deployment <deploy>

```

#### 네트워크 문제가 발생한 경우

```sh
sudo systemctl restart kubelet   <-- worknode 에서
```

```
kubelet 종료
↓
kubelet 재시작
↓
API Server와 재연결
↓
Node 상태 재등록
↓
Pod 상태 다시 동기화
```

## mysql 실행

```sh
# mysql pod 실행
kubectl apply -f pv.yaml
kubectl apply -f pvc.yaml
kubectl apply -f mysql_jdbctest_deployment.yaml
kubectl apply -f mysql_service_nodeport.yaml

# 상태 확인
kubectl get node -o wide
kubectl get svc
kubectl get  pod -o wide
kubectl  logs -f mysql-58b988ddbb-6w22d
kubectl describe  pod mysql-58b988ddbb-6w22d

#  calico  상태 확인
kubectl get pods -n kube-system -o wide
kubectl get pods -n kube-system -l k8s-app=calico-node -o wide
kubectl logs -n kube-system calico-node-8rldt

# worknode kubectl 재시작, clico 재시작, mysql 재시작
sudo systemctl restart kubelet   <-- worknode 에서
kubectl rollout restart daemonset calico-node -n kube-system
kubectl rollout restart deployment mysql
```
